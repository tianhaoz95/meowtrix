User Guide

Everything you need to install Meowtrix and get productive in the workspace.

Overview

Meowtrix is a browser-based remote workspace. The UI is a tiling pane layout: each pane holds tabs, and each tab is either a terminal (a real PTY-backed shell, rendered with xterm.js) or an embedded browser. A single small Node server hosts the frontend and multiplexes the terminal sessions over a WebSocket.

You run it on a host machine (your laptop, a home server, a VPS) and reach it from any browser on the network — or from anywhere via a tunnel. Because the shells live on the server, you can close the tab, switch from laptop to phone, or ride out a dropped connection without losing your running processes.

💡

Want to poke around first? The live demo runs entirely in your browser — no install required (terminals become a JavaScript REPL there). See Trying the demo.

Installation

Prerequisites

Meowtrix runs on the host machine (macOS or Linux) — the devices you connect from only need a browser.

  • Zero-Dependency Install (Recommended): No dependencies required. The installer automatically downloads the pre-packaged Node.js runtime and compiled native modules for your platform.
  • Source-Based Install:
    • Node.js 18+ and npm — runs the server
    • git — clones the repo
    • A C/C++ build toolchain — to compile node-pty natively:
      • macOS: Xcode Command Line Tools (xcode-select --install)
      • Linux: build-essential (or gcc/make) and python3
🪟

The one-line installer and the --service auto-start mode support macOS and Linux only — Windows is not supported.

Zero-Dependency Binary Install (Recommended)

The fastest path is the one-line installer. If git or node are not detected on your system, it automatically falls back to downloading a precompiled release. Run:

curl -fsSL https://raw.githubusercontent.com/tianhaoz95/meowtrix/main/install.sh | bash

You can force this zero-dependency binary install by passing the --binary flag:

curl -fsSL https://raw.githubusercontent.com/tianhaoz95/meowtrix/main/install.sh | bash -s -- --binary

Source-Based Install

If you prefer to compile from source and have all prerequisites installed, you can clone the repository and run the installer:

git clone https://github.com/tianhaoz95/meowtrix.git ~/.meowtrix/app
cd ~/.meowtrix/app
./install.sh

Or run directly without installation:

git clone https://github.com/tianhaoz95/meowtrix.git
cd meowtrix
npm install      # node-pty compiles natively
npm start

Install as a background service

Add --service to auto-start Meowtrix on login and restart it if it crashes (launchd on macOS, systemd on Linux):

curl -fsSL https://raw.githubusercontent.com/tianhaoz95/meowtrix/main/install.sh | bash -s -- --service
📦

Updating is as simple as re-running the installer (it pulls the latest changes). In addition, binary installations support one-click auto-updates directly from the browser's update banner.

Uninstall

To fully uninstall Meowtrix, stop the service (if running), remove the service configuration, and delete the installation files.

macOS:

# Stop and remove the launchd service
launchctl unload ~/Library/LaunchAgents/com.meowtrix.plist 2>/dev/null || true
rm -f ~/Library/LaunchAgents/com.meowtrix.plist

# Remove launcher and files
rm -f ~/.local/bin/meowtrix
rm -rf ~/.meowtrix

Linux:

# Stop and disable the systemd service
systemctl --user stop meowtrix 2>/dev/null || true
systemctl --user disable meowtrix 2>/dev/null || true
rm -f ~/.config/systemd/user/meowtrix.service
systemctl --user daemon-reload

# Remove launcher and files
rm -f ~/.local/bin/meowtrix
rm -rf ~/.meowtrix

Running it

Start the server (defaults to port 9123, override with the PORT env var):

meowtrix            # if installed via the installer
# or, from source:
npm start
PORT=8080 npm start

Then open http://localhost:9123 in your browser. You'll land in a single pane with one terminal, ready to type.

By default Meowtrix binds to 127.0.0.1 (localhost only), so it is not reachable from other machines on your network — see Reaching it remotely to change that safely.

Reaching it remotely

Meowtrix hands whoever can reach it a real shell on the host, so it binds to 127.0.0.1 (localhost only) by default. There are two safe ways to use it from another device.

1. SSH tunnel (recommended)

Leave the default localhost binding and forward the port over SSH from your client machine:

ssh -L 9123:localhost:9123 <user>@<host>
# then open http://localhost:9123 in your local browser

This keeps Meowtrix off the network entirely — only your authenticated SSH session can reach it. Tunnels like ngrok http 9123 work the same way, since they connect to localhost on the host.

2. Bind to the network

To reach Meowtrix directly from other devices on a trusted network, bind to all interfaces:

meowtrix --network          # or: meowtrix --host 0.0.0.0
# from source:
HOST=0.0.0.0 npm start

Then open http://<host-ip>:9123 from any device. Use --host <addr> (or the HOST env var) to bind a specific interface instead.

🔐

There is no built-in authentication. --network exposes a shell to anyone who can reach the port — only do this on a network you trust, or front it with your own auth/firewall. The SSH-tunnel option above avoids this entirely.

Panes & tabs

The workspace is a tree of panes. Each pane has a tab strip; open as many tabs as you like inside a pane.

  • Split a pane with the toolbar's Split V (side by side) or Split H (top/bottom) buttons.
  • Resize by dragging the divider between panes. Sizes are proportional, so they hold up when you resize the window or move between screens.
  • Add a tab with the + on the tab strip, then pick Terminal or Browser.
  • Reorder / move tabs by dragging them — within a pane or across panes (works with mouse and touch).
  • Close a tab with its ✕, or close the whole pane with the toolbar's Close button.

Splitting again in the same direction adds an equal sibling rather than nesting deeper, so layouts stay flat and predictable.

Terminals

A terminal tab is a full PTY-backed shell — your real shell, with colors, TUIs, and job control. The key property is that the shell lives on the server, not in the browser tab:

  • Refresh the page, and the terminal reattaches to the same running shell with its recent output replayed.
  • Close the browser, walk away, come back on another device — the process kept running.
  • A dropped connection auto-reconnects (about once a second) and resumes the live stream.

Double-click (or double-tap) anywhere in a terminal to send Tab — handy for autocomplete without reaching for the key.

⚠️

Terminals are in-memory on the server. Restarting the Meowtrix server kills all shells — but your layout is saved and will be rebuilt.

Browser panes

A browser tab is an embedded mini-browser with an address bar, back/forward, reload, and an "open in a new window" (↗) button. Pages are fetched through a built-in server-side proxy that strips frame-blocking headers, so many sites that normally refuse to be embedded will render right in the pane.

Some sites — Google, sign-in flows, heavily bot-protected pages — still won't embed. Use ↗ to open those in a real browser window. You can set a default homepage for new browser tabs in Settings.

Code editor & Git

Meowtrix comes with a built-in code editor based on Monaco (the same core that powers VS Code). You can open a folder directly via the tab picker (+ → Code Editor), the command palette, or by typing mtx code /path/to/dir in any terminal.

  • File Tree Sidebar: Browse the directory structure of the host machine and click to open files. Unsaved changes are marked with a dot indicator.
  • Markdown Preview: Opening a .md file reveals a Preview button in the editor. Toggle it to view a live, side-by-side formatted preview. A quick-copy button is available on code blocks inside the preview.
  • HTML Preview: View your static HTML files directly as preview tabs.
  • Source Control: If the opened folder is a git repository, a Source Control tab appears in the sidebar. You can review staged/unstaged changes, view side-by-side diffs, stage/unstage files, and perform commits, pushes, and pulls.
  • AI-Powered Commit Messages: Click the AI generate button next to the commit message box to automatically write a descriptive commit message based on your staged diffs.

Broadcast input

Toggle the toolbar's 📡 Broadcast button to mirror your keystrokes to every visible terminal at once — like iTerm2 broadcast or tmux synchronize-panes. Great for running the same command across several hosts side by side. Toggle it off to return to typing into just the focused terminal.

Scheduled Enter presses (⏰)

When working with AI coding agents or long-running commands, you may hit usage quotas or time constraints. Scheduled Enter lets you type a command and schedule its execution for later.

Click the ⏰ button in the toolbar or open it via the command palette to set a delay (e.g. 5h 30m) or a specific clock time. The target terminal will display a blurred countdown overlay and swallow input until the timer fires, writing an Enter character (\r) to the PTY. Because the timer lives server-side next to the shell processes, it will fire even if you close your browser tab or disconnect.

Command palette

Press ⌘K (macOS) or Ctrl/⌘ + Shift + P anywhere to open a fuzzy launcher for every workspace action — no need to remember individual shortcuts or hunt through the toolbar. You can also click the ⌘ Commands button in the toolbar.

  • Type to filter; the list narrows as you go (fuzzy, so "spv" finds "Split pane vertically").
  • ↑/↓ to move, Enter to run, Esc to dismiss.

Commands include splitting panes, opening terminal or browser tabs, switching between tabs and panes, toggling broadcast input, switching themes, uploading a file, and opening settings.

⌨️

⌘K and Ctrl/⌘+Shift+P are used (rather than Ctrl+K or Ctrl+P) so they don't clash with the readline editing keys your shell uses inside the terminal.

Sessions & multiple devices

Only one client is the active session at a time. If you open Meowtrix in a second browser or on another device, the newest one takes over and the others show a "Session active elsewhere" overlay with a Move session here button.

This is coordinated by the server (not just within one browser), so it works across different browsers and devices. When you move the session, your layout follows — the workspace is rebuilt and reconnects to the still-running shells.

On mobile

Meowtrix is built to be usable from a phone:

  • An on-screen key bar sits above the soft keyboard with the keys terminals need most (Esc, Tab, arrows, and modifiers).
  • Ctrl / Alt / Cmd are "sticky" — tap one, then tap a key to combine them (e.g. Ctrl + C), since mobile keyboards don't emit reliable key events.
  • Double-tap the terminal to trigger autocomplete (Tab).

On-device Chat Pet

Mochi is a tiny animal companion that wanders around your workspace. You can click on Mochi to open a chat dialog. Mochi replies instantly, keeping you company as you build.

  • On-device AI: All conversation is processed completely locally in your browser. Replies are generated by Chrome's on-device Gemini Nano model (via the Prompt API). No network requests are made, keeping your chats completely private.
  • Customizable Faces: Go to settings to choose from 12 different faces for your pet: Cat, Dog, Fox, Bunny, Panda, Penguin, Frog, Chick, Ghost, Robot, Dragon, or Unicorn.
  • Wander Controls: You can adjust Mochi's speed (1 for lazy, up to 10 for zoomies) or enable "Stay" to keep Mochi pinned to a specific location (simply drag Mochi anywhere on the screen).
ℹ️

The Chat Pet requires Google Chrome with the experimental Prompt API / LanguageModel enabled. If your browser does not support it, the pet toggle in Settings will be disabled.

Keystroke combo FX

Celebrate your coding streaks! Keystroke Combo FX tracks your typing speed and streak length to drive dynamic, gamified visual effects in real time.

  • Tiers: As you type, your combo builds up through tiers: HEATING UP, ON FIRE, BLAZING, INFERNO, and MELTDOWN.
  • Visual Feedback: A heat-tinted combo readout counts up in the bottom-right corner. Keystrokes produce particle bursts, the screen edges glow, and reaching the highest tiers triggers a screen-shake effect.
  • Toggleable: If you find the effects distracting, you can toggle them off in the Settings panel.

Self-updates

Meowtrix handles updates directly from the web interface. The server periodically checks its git clone upstream status in the background. If a newer version is available, a non-intrusive update banner will appear at the top of the workspace.

Clicking Update & Restart pulls the latest code (re-compiling native dependencies if needed) and restarts the server process. This is kept user-triggered because restarting the server terminates active PTY terminals, letting you choose the perfect moment to update.

Trying the demo

The live demo is the full Meowtrix UI running with no server. Because there's no host machine, terminals are replaced by an in-browser JavaScript REPL, and your layout and settings persist to the browser's local storage.

It's the quickest way to feel the panes-and-tabs workflow. Note that the embedded browser there loads pages directly (no proxy), so only sites that allow embedding will appear. For real terminals and the full proxy, install Meowtrix.
